Filosofía de Diseño

AsistenteHogar es una aplicación móvil premium de cocina y gestión de despensa que combina minimalismo cálido con interactividad nativa de iOS. Cada componente respeta la identidad visual "Tierra Cálida" mientras proporciona feedback táctil, animaciones fluidas y una accesibilidad irreprochable (WCAG AA).

Principios nucleares:
  • Calidez sin fricción: Tonos marrones y crema que confortan, interacciones que aceleran.
  • Accesibilidad primero: Contraste mínimo AA, tamaños de toque ≥44px, estados enfocables por teclado.
  • Micro-animaciones: Spring physics (no CSS lineales), haptic feedback guides, shared elements.
  • Sin slop: Nada de gradientes fríos, blanco puro, emoji, NativeWind, o fuentes customizadas.

Componentes

Button

✓ Correcto

  • Padding vertical 13-16px, horizontal 20px
  • Border-radius: 999px (pill)
  • Transiciones suaves (0.2s ease)
  • Estados: normal, hover, active, focus, disabled
  • Ícono + texto centrados con gap 8px

✗ Incorrecto

  • Bordes cuadrados o radius pequeños
  • Gradientes fríos o multi-color
  • Sin transiciones o transiciones lineales
  • Touch target <44px
  • Sombras negras puras

Input

✓ Correcto

  • Border 1.5px de color border
  • Focus: outline 3px offset con 0.1 alpha
  • Placeholder: color faint (#AE9B87)
  • Padding: 13px × 16px
  • Error: border danger, soft danger bg

✗ Incorrecto

  • Borders gruesos o colores genéricos
  • Sin hint text para validación
  • Loading spinner sin icono
  • Altura <44px

Card

✓ Correcto

  • Border-radius: 32px
  • Sombra: 0 6px 14px rgba(..., 0.07)
  • Hover: elevar +2px, sombra mayor
  • Tintes semánticos disponibles

✗ Incorrecto

  • Fondos blancos puros
  • Bordes visibles innecesarios
  • Sin transiciones al interactuar

Skeleton Loader

Usar cuando datos cargan de IA o API. En React Native no usar gradientes CSS ni animaciones CSS. Implementar un pulso de opacidad con Animated.loop:

const pulse = useRef(new Animated.Value(0.3)).current;

useEffect(() => {
  Animated.loop(
    Animated.sequence([
      Animated.timing(pulse, { toValue: 0.7, duration: 800, useNativeDriver: true }),
      Animated.timing(pulse, { toValue: 0.3, duration: 800, useNativeDriver: true }),
    ])
  ).start();
}, []);

// Fondo: colors.cardAlt (#F9F4EC) o colors.brandSoft (#F6EDE3)
<Animated.View style={{ opacity: pulse, backgroundColor: colors.cardAlt }} />

Usar useNativeDriver: true siempre para que la animación corra en el hilo nativo y no bloquee el JS thread.

Micro-Animaciones

Motion & Haptic

Spring Physics

Para React Native: Usar Animated.spring() con useNativeDriver: true.

Timing sugerido: stiffness 120, damping 14 (snappy pero no jarring).

Casos: Presionar botones, escalar cards, desplazar bottom sheets.

Haptic Feedback

iOS: Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium) en toques principales.

Casos:

  • Tap en botón primario → Medium impact
  • Swipe en card → Light impact
  • Confirmación de acción → Success notification
  • Error → Warning notification

Shared Element Transitions

Cuando navegas de lista de recetas a detalle, la tarjeta "viaja" de su posición original a fullscreen. Usar Reanimated 3 Shared Transitions: añadir la prop sharedTransitionTag="recipe-card-{id}" al Animated.View tanto en la lista como en el detalle.

// Lista
<Animated.View sharedTransitionTag={`recipe-card-${item.id}`}>

// Detalle
<Animated.View sharedTransitionTag={`recipe-card-${recipe.id}`}>

Tamaños de toque

  • Botones: mínimo 44×44px (recomendado 48×48px)
  • Inputs: altura 44px mínimo, ancho responde
  • Chips/badges: 32px altura
  • Toggles: 52×32px (thumb 28×28px)

Teclado & Focus

  • Todos los botones/inputs: outline: 2px solid #8B5E3C; outline-offset: 2px;
  • Tab order lógico (izq→der, arriba→abajo)
  • Aria-labels en iconos sin texto
  • Aria-live para toasts y notificaciones

Labels & Hints

  • Inputs siempre con <label>
  • Hint text debajo (color muted, 12px)
  • Error hints: color danger, descriptivo

Anti-Patrones — NUNCA

Restricciones del proyecto
  • ❌ NativeWind / Tailwind: Desinstalado. Todo CSS custom en StyleSheet.
  • ❌ Gradientes fríos: Púrpura, azul frío, blanco-a-gris. Solo tonos tierra.
  • ❌ Blanco puro (#FFFFFF): Usar crema warmth (#FFFDF8).
  • ❌ Negro puro (#000000): Usar marrón oscuro (#2C1C0E).
  • ❌ Fuentes customizadas pesadas: Solo SF Pro (iOS) / Roboto (Android).
  • ❌ Emoji en UI: Iconografía limpia vía @expo/vector-icons.
  • ❌ Sombras negras: Usar rgba(44, 28, 14, ...).
  • ❌ Borders gruesos genéricos: 1-1.5px, color #D9CCBC / #EAE0D3.
